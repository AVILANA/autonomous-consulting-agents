# Step 7: Due Diligence — Kill My Thesis

Read all prior outputs before completing this step.

## Outputs
- `outputs/{company}/due_diligence.md`
- `outputs/{company}/discovery_questions.md`

## Instructions
1. Challenge the pursuit thesis and identify the strongest counterarguments.
2. Produce validation tests for each counterargument.
3. Recommend conditional thesis adjustments.
4. Create 15 prioritized discovery questions ranked by impact.
5. Provide an overall confidence assessment: Low / Medium / High.

## DATA FRESHNESS RULE — MANDATORY
For every KPI and every data point used in any output:

1. Always use the most recent available figure. If FY2025 data exists for a metric (even from a press release or earnings transcript), use FY2025. Only fall back to FY2024 if the specific metric was not disclosed in any FY2025 source.
2. Every number used in a counterargument or validation test must show its fiscal year. If the thesis rests on FY2024 data but FY2025 data exists that changes the picture, that is itself a counterargument — flag it.
3. Never construct a counterargument based on stale data without checking whether more recent data changes the counterargument's validity.
4. When assessing confidence, factor in data freshness: a finding based on FY2023 data with no FY2024 or FY2025 confirmation should reduce confidence, not maintain it.
5. Discovery questions must be dated: if a gap could have been closed by more recent disclosures you haven't found, flag this as a research gap.
