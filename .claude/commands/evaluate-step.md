# Step Output Evaluator

You are a quality assurance reviewer. Read memory.md for all quality standards. Then read the output files that were just produced.

Evaluate against EVERY standard in memory.md. Specifically check:

1. Value levers: are they EXACTLY Productivity, Efficiency, Quality, Velocity, Growth? Any renamed or added?
2. Body vs Brain: does body mean physical network only? Is process design correctly classified as brain?
3. Financial data: is there full P&L with operating expenses, not just SG&A? Are COGS logistics components highlighted?
4. Peer set: geographic diversity across at least 2 of 3 regions? No major public competitor missing?
5. Supply chain: deep dive by individual area? Velocity and delivery speed assessed?
6. Delivery and SLAs: fulfillment speed, network responsiveness, body readiness score present?
7. Language: simple, executive-friendly? Every acronym explained first time?
8. Claims: every claim tagged as FACT, INFERENCE, or ASSUMPTION?
9. Technology: implementation landscape searched? TMS, WMS, ERP, AI platforms checked?
10. Executive summary: clean table format with 3 bold bullet points per step? Post-due-diligence view?
11. Benchmark table: includes Total Operating Cost not just SG&A? Footnotes on where supply chain costs sit?

For each check, mark PASS or FAIL with a brief explanation.

If ANY check is FAIL:
- List exactly what failed
- Rewrite the output files fixing ONLY the failed items
- Save the corrected version overwriting the original

If ALL checks PASS:
- Write ALL CHECKS PASSED

Save the evaluation report to the company output folder as evaluation_report.md
