# Agent Memory
This file is read by every step at the start of execution. It contains accumulated learnings, corrections, and quality standards from previous executions and user feedback.

## Quality Standards (learned from feedback)
- Value levers must be EXACTLY: Productivity, Efficiency, Quality, Velocity, Growth. Never rename or substitute.
- Body = physical network (warehouses, suppliers, routes, DCs). Brain = how you operate within it (processes, planning, systems, AI). If you can change it without physically moving or closing something, it is BRAIN.
- Peer sets must have geographic diversity: Americas, Europe, Asia-Pacific. Always verify with a web search.
- Financial snapshots must include full P&L: Revenue, COGS, Gross Profit, Operating Expenses (not just SG&A), Operating Income, EBITDA, Net Income, CapEx, Debt, Working Capital.
- SG&A alone does not capture operating costs. Many logistics and fulfillment costs sit in COGS. Always look for total operating cost structure, not just SG&A.
- Benchmark tables MUST include Total Operating Cost % Revenue, not just SG&A. Note where logistics costs sit in each company P&L (COGS vs SG&A). SG&A-only comparisons are misleading.
- Step 4 MUST include a Delivery Speed and Fulfillment Assessment with: delivery times, SLAs, ship-from-store, microfulfillment, dark stores, click-and-collect, DC count and locations, automated DC capabilities, last mile model. Must produce a Body Readiness Score. This section is NOT OPTIONAL.
- Executive summary must be a CLEAN table with exactly 3 bullet points per step, max 15 words each, key insight in bold. No paragraphs.
- Executive summary must reflect POST-due-diligence view, not pre-challenge optimistic claims.
- Auto-evaluator runs after each step checking all quality standards.
- Supply chain analysis must cover: Procurement, Sourcing, Planning, Warehousing, Inbound Transport, Outbound Transport, Last Mile, Transport Planning and Execution, Freight Audit, Returns.
- Always search for delivery times, SLAs, fulfillment speed, and network responsiveness. Assess whether the physical network (body) can keep up with the speed the brain enables. Look for microfulfillment, ship-from-store, dark store signals.
- Always search for technology implementations in progress: TMS, WMS, ERP, AI platforms, and which partners are implementing them.
- Use simple language. Explain every acronym the first time. No jargon without explanation.
- The final memo must reflect the stress-tested thesis from due diligence, not the pre-due-diligence optimistic version.
- No 12-week roadmap in the memo. Focus on opportunities with quantified benefits and validation questions.
- Include an executive summary at the start where each analytical step summarizes its top 3 findings.
- Due diligence must produce prioritized discovery questions ranked by decision impact.
